SECTION A: PROJECT CONTEXT  (always include this)
────────────────────────────────────────────────────────────────────────────────

I'm working on an internal enterprise portal called IPI Portal.

Tech stack:
  - Next.js 16 App Router, React 19, TypeScript 5
  - Tailwind CSS 4, Framer Motion, dnd-kit, Lucide React
  - Microsoft SQL Server via the `mssql` driver
  - NTLM Windows authentication decoded in lib/auth.ts
  - Active Directory enrichment via PowerShell (Get-ADUser)
  - AES-256-CBC encryption (Node.js crypto) for password fields
  - JWT sessions via `jose` library (HS256, 1-hour expiry, httpOnly cookie)
  - bcryptjs for admin password hashing
  - xlsx library for Excel import/export (client-side)

Project structure:
  - app/page.tsx              Server component entry point
  - app/ClientPage.tsx        Entire UI in one large client component (~1800 lines)
  - app/layout.tsx            Root layout, loads fonts
  - app/api/sections/         Main CRUD + visibility filtering + encryption
  - app/api/auth/             Admin login → bcrypt check → JWT cookie
  - app/api/logout/           Clears admin_token cookie
  - app/api/verify-pin/       Server-side PIN check for revealing passwords
  - app/api/phonebook/        Contact directory
  - app/api/messages/         System announcements
  - app/api/user/             NTLM identity + AD lookup (input validated)
  - app/api/settings/         Holiday decoration toggles (JSON file)
  - lib/auth.ts               NTLM decoder, AD enrichment, user cache,
                              createAdminToken(), verifyAdminToken()
  - lib/db.ts                 MSSQL connection pool (env vars only, no fallbacks)

Database tables (SQL Server):
  Sections        (Id, Title, Color, SchemaDef JSON)
  Items           (Id, SectionId, Data JSON, visibility)
  SystemMessages  (id, text, date)
  PhonebookConfig (Id=1, SchemaDef JSON)
  PhonebookData   (Id=1, Content JSON blob)
  AuthorizedUsers (Username) — current: itayc, gal, michaelg, idant
  GlobalSettings  (SettingKey, SettingValue) — EditPassword = bcrypt hash

Security status (completed):
  - AES key in ENCRYPTION_KEY env var (not hardcoded)
  - Admin password bcrypt hashed in DB
  - PowerShell injection protected with regex validation
  - DB credentials in env vars only
  - Password reveal PIN checked server-side
  - Admin list in NEXT_PUBLIC_ADMIN_USERS env var
  - JWT httpOnly cookie for admin sessions
  - All POST /api/sections require valid JWT (401 if missing)

Key conventions:
  - Hebrew strings are used throughout the UI — preserve them exactly
  - isEditMode controls admin UI — gated by JWT cookie on the server
  - Item visibility is a comma-separated string: 'הכל' = everyone,
    otherwise department names / usernames / job titles / AD groups
  - Password fields are AES encrypted in SQL, decrypted before sending to client
  - Phonebook uses draft/commit pattern: draftPhonebook (edit) vs phonebookData (saved)
  - All API routes return HTTP 200 even on auth failure (to avoid browser dialogs)
  - Direction is RTL (dir="rtl"), layout is right-to-left
  - Admin cookie: name = admin_token, httpOnly, sameSite=strict, maxAge=3600

JWT verification (server-side, in any API route):
  import { verifyAdminToken } from '@/lib/auth';
  const cookieHeader = request.headers.get('cookie') || '';
  const adminToken = cookieHeader.match(/admin_token=([^;]+)/)?.[1] || '';
  const isValidAdmin = await verifyAdminToken(adminToken);
  if (!isValidAdmin) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
