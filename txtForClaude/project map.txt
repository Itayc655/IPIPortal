1. WHAT THIS IS
────────────────────────────────────────────────────────────────────────────────

Internal enterprise portal for the IPI organization. A single-page app that
serves as a centralized hub for:

  - Browsing organizational knowledge cards (products, procedures, links)
  - Role-based access control tied to Windows Active Directory (AD)
  - A company phonebook with department filtering and Excel import/export
  - System-wide announcements (scrolling ticker)
  - Secure storage of credentials (usernames + encrypted passwords)
  - Holiday and memorial day visual decorations (admin-toggled)

One public route: GET /
Everything else is API routes under /api/*.
The entire UI lives in a single file: app/ClientPage.tsx

2. TECH STACK
────────────────────────────────────────────────────────────────────────────────

  Runtime         Next.js 16 (App Router), React 19, TypeScript 5
  Styling         Tailwind CSS 4
  Animations      Framer Motion
  Drag & Drop     @dnd-kit/core + @dnd-kit/sortable
  Icons           Lucide React
  Database        Microsoft SQL Server via `mssql` driver
  Auth            NTLM (Windows Integrated Auth) decoded in lib/auth.ts
  AD Enrichment   PowerShell → Get-ADUser (runs on the Next.js server)
  Encryption      Node.js built-in `crypto` module, AES-256-CBC
  JWT Sessions    `jose` library (HS256, 1-hour expiry)
  Password Hash   bcryptjs
  Excel           xlsx library (client-side parse + generate)

4. LAYER-BY-LAYER BREAKDOWN
────────────────────────────────────────────────────────────────────────────────

BROWSER / CLIENT (app/ClientPage.tsx)
  - Admin detection: reads NEXT_PUBLIC_ADMIN_USERS env var.
  - Dev mode: NEXT_PUBLIC_DEV_ADMIN=true bypasses AD check locally.
  - On login: POST /api/auth → JWT cookie set by server automatically.
  - On logout: POST /api/logout → cookie cleared, isEditMode = false.
  - All write requests include the JWT cookie automatically (httpOnly).

NEXT.JS SERVER (lib/)
  - lib/auth.ts: getCurrentUser() + createAdminToken() + verifyAdminToken()
  - lib/db.ts: reads DB_* env vars, throws if any are missing.

API ROUTES (/api/*)
  - auth:         bcrypt password check → JWT cookie
  - logout:       clears cookie
  - sections:     JWT verified on ALL POST requests (401 if invalid)
                  editMode only works if JWT is valid
  - verify-pin:   checks REVEAL_PIN env var server-side
  - user:         validates input regex before PowerShell call
  - phonebook, messages, settings: standard GET/POST

DATA STORES
  - SQL Server:   primary store for all data
  - .env:         credentials and secrets
  - holiday-settings.json: decoration state
  - Active Directory: read-only, via PowerShell

5. DATABASE SCHEMA (SQL Server)
────────────────────────────────────────────────────────────────────────────────

  Sections        Id, Title, Color, SchemaDef JSON
                  Colors: red|blue|green|purple|orange|teal|indigo|pink
                  Field types: text|textarea|link|password|file|folder

  Items           Id, SectionId, Data JSON, visibility
                  visibility: comma-separated list or 'הכל' (everyone)
                  Password fields in Data are AES-256 encrypted.

  SystemMessages  id (timestamp string), text, date (DD/MM/YYYY)

  PhonebookConfig Id=1, SchemaDef JSON (column definitions)
  PhonebookData   Id=1, Content JSON (all employee rows)

  AuthorizedUsers Username (AD sAMAccountName, lowercase)
                  Current: itayc, gal, michaelg, idant

  GlobalSettings  SettingKey, SettingValue
                  EditPassword → bcrypt hash (local) / plain text (production)


────────────────────────────────────────────────────────────────────────────────
6. AUTHENTICATION & AUTHORIZATION FLOW
────────────────────────────────────────────────────────────────────────────────

  1. NTLM header → lib/auth.ts decodes username
  2. Fallback: IP-to-username cache → guest user
  3. AD enrichment: PowerShell Get-ADUser → department, title, groups (60min cache)
  4. Admin login: POST /api/auth → bcrypt check → JWT cookie (httpOnly, 1h)
  5. Writes: extract admin_token cookie → verifyAdminToken() → 401 if invalid
  6. Logout: POST /api/logout → cookie cleared


────────────────────────────────────────────────────────────────────────────────
7. VISIBILITY / PERMISSIONS SYSTEM
────────────────────────────────────────────────────────────────────────────────

Server-side filtering in /api/sections GET.
Item is shown if ANY of these match:
  - visibility = 'הכל'
  - editMode=true AND JWT is valid
  - user's department in allow-list
  - user's username in allow-list
  - user's job title in allow-list
  - any of user's AD groups in allow-list

Items that don't pass are never sent to the browser.


────────────────────────────────────────────────────────────────────────────────
8. DATA ENCRYPTION
────────────────────────────────────────────────────────────────────────────────

AES-256-CBC. Key from ENCRYPTION_KEY env var (32 chars).
Only `password` sub-field of 'password' type fields is encrypted.
Format in SQL: "ivHex:cipherHex"
Decrypted server-side before sending to client.

Admin password: bcrypt hash in GlobalSettings.EditPassword.
PIN: stored in REVEAL_PIN env var, checked server-side.

9. KEY STATE VARIABLES (ClientPage.tsx)
────────────────────────────────────────────────────────────────────────────────

  systemMessages        array     [{ id, text, date }]
  showHolidayDecor      boolean   Independence Day bunting
  memorialDayType       null | 'holocaust' | 'idf'
  unlockedPasswords     Record    fieldKey → secondsRemaining countdown
  pendingAction         object    Deferred password action until PIN verified


────────────────────────────────────────────────────────────────────────────────
10. EDIT MODE FLOW
────────────────────────────────────────────────────────────────────────────────

  Enter: lock button → login modal → POST /api/auth → JWT cookie → isEditMode=true
  Active: all writes include cookie, server verifies on every POST
  Exit: POST /api/logout → cookie cleared → isEditMode=false


────────────────────────────────────────────────────────────────────────────────
11. PHONEBOOK FLOW
────────────────────────────────────────────────────────────────────────────────

  Draft/commit: phonebookData (saved) ↔ draftPhonebook (edit mode working copy)
  Save: draftPhonebook → phonebookData + POST /api/phonebook
  Birthday: filters by current month, format DD.MM.YYYY or DD/MM/YYYY
  Excel import: client-side parse → validate → draftPhonebook → SQL


────────────────────────────────────────────────────────────────────────────────
12. HOLIDAY / MEMORIAL DAY DECORATIONS
────────────────────────────────────────────────────────────────────────────────

  State loaded from /api/settings on page mount (persists across reloads).
  Saved to holiday-settings.json on every toggle.
  Independence Day: IndependenceDayDecor.tsx — Israeli flag bunting
  Memorial Day: MemorialDayBanner.tsx — fixed black bar, modes: holocaust|idf

16. DEPLOYMENT NOTES
────────────────────────────────────────────────────────────────────────────────

Environments:
  Local:       C:\IPIPortal          (DB_INSTANCE=SQLEXPRESS01)
  Production:  C:\Users\itayc\Desktop\product-hub  (DB_INSTANCE=SQLEXPRESS)

Deploy sequence:
  1. git pull origin main
  2. npm install
  3. npm run build
  4. pm2 restart all
  5. pm2 logs --lines 20  →  verify "DB Pool Connected Successfully"
  6. Test: portal loads, cards, phonebook, admin login, add/delete item

Rollback:
  git revert HEAD && npm run build && pm2 restart all

After first deploy with new security code:
  Create scripts/hash-password.mjs (see SEC-2 notes), run it once,
  delete it immediately. This hashes EditPassword in production DB.

Files required on server, not in git:
  .env  (with all variables from section 13)
  holiday-settings.json  (auto-created on first settings save)
